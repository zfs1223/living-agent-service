# Compliance Check

> 合规检查技能，自动化合规审查、政策解读、风险预警

## 一、技能描述

合规检查技能使法务部数字员工能够执行企业合规相关任务，包括：
- 法规政策跟踪
- 合规风险识别
- 合规审查流程
- 合规培训管理

## 二、适用编制

| 编制代码 | 职位 | 主要用途 |
|---------|------|---------|
| L02 | 合规专员 | 合规检查、政策解读 |
| L01 | 合同审查员 | 合同合规审查 |

## 三、触发词

- 合规、compliance、合规检查
- 政策解读、法规
- 风险预警、合规风险
- 合规培训、合规审查

## 四、功能模块

### 4.2 合规审查

| 功能 | 说明 |
|------|------|

| 合同审查 | 合同条款合规检查 |
| 流程审查 | 业务流程合规检查 |
| 数据合规 | 数据处理合规检查 |

### 4.3 风险管理

| 功能 | 说明 |
|------|------|
| 风险识别 | 识别合规风险点 |
| 风险评估 | 评估风险等级 |

## 五、使用示例

### 5.1 合规审查

```
用户: 审查新员工手册的合规性

技能执行:
1. 解析员工手册内容
2. 匹配劳动法规要求
3. 识别合规风险点
4. 生成审查报告
5. 提出修改建议
```

### 5.2 法规影响分析

```
用户: 分析新《个人信息保护法》对公司业务的影响

技能执行:
1. 解析法规要点
2. 匹配公司业务场景
3. 识别受影响流程
4. 评估整改需求
5. 生成影响分析报告
```

### 5.3 合规培训

```
用户: 组织数据安全合规培训

技能执行:
1. 确定培训对象
2. 准备培训材料
3. 安排培训时间
4. 发送培训通知
5. 跟踪培训完成情况
```

## 七、合规风险等级

| 等级 | 说明 | 处理时限 |
|------|------|---------|
| 🔴 高风险 | 可能导致行政处罚或诉讼 | 立即整改 |
| 🟡 中风险 | 可能导致经济损失或声誉影响 | 30天内整改 |
| 🔵 低风险 | 轻微不合规 | 90天内整改 |

## 八、合规报告模板

```markdown
# 合规审查报告

## 基本信息
- 审查对象: [合同/制度/流程]
- 审查日期: 2026-03-13
- 审查人: L02

## 审查结果
- 总体评级: ⚠️ 需改进
- 高风险项: 2
- 中风险项: 3
- 低风险项: 5

## 风险详情

### 🔴 高风险项
1. **劳动合同未及时签订**
   - 问题描述: 新员工入职超过1个月未签订劳动合同
   - 法规依据: 《劳动合同法》第10条
   - 整改建议: 立即补签劳动合同
   - 整改期限: 立即

### 🟡 中风险项
1. **加班时长超限**
   - 问题描述: 部分员工月加班时长超过36小时
   - 法规依据: 《劳动法》第41条
   - 整改建议: 调整排班，控制加班时长
   - 整改期限: 30天

## 整改跟踪
| 风险项 | 责任人 | 整改状态 | 截止日期 |
|--------|--------|---------|---------|
| 劳动合同签订 | HR | 进行中 | 2026-03-15 |
| 加班时长控制 | 各部门 | 待开始 | 2026-04-13 |
```

## 九、自动化规则

| 触发条件 | 自动动作 |
|---------|---------|
| 新法规发布 | 分析影响、发送通知 |
| 发现高风险项 | 立即通知法务主管 |
| 整改超期 | 升级提醒 |
| 季度末 | 生成合规报告 |

```yaml
    low_risk: weekly

    frequency: quarterly

```
