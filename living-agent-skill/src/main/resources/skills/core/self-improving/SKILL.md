# Self Improving

> 自我改进技能，捕获学习、错误和纠正以实现持续改进

## 一、技能描述

自我改进技能使数字员工能够从错误中学习并持续改进，包括：
- 错误捕获和记录
- 纠正学习
- 经验积累
- 模式优化

## 二、适用编制

| 编制代码 | 职位 | 主要用途 |
|---------|------|---------|
| T01 | 代码审查员 | 代码错误学习 |
| T07 | 安全工程师 | 安全问题学习 |
| 所有 | 所有员工 | 持续改进 |

## 三、触发词

- 错误、error、失败
- 改进、improve、优化
- 学习、learn
- 纠正、correct

## 五、使用示例

### 5.1 错误捕获

```
场景: 执行命令失败

技能执行:
1. 捕获错误信息
2. 分析错误类型
3. 查找历史解决方案
4. 应用解决方案
5. 记录学习结果
```

### 5.2 纠正学习

```
场景: 用户纠正了输出

技能执行:
1. 记录原始输出
2. 记录用户纠正
3. 分析差异原因
4. 更新行为模式
5. 应用到未来任务
```

### 5.3 经验积累

```
场景: 成功完成任务

技能执行:
1. 记录成功步骤
2. 提取成功模式
3. 关联任务类型
4. 更新最佳实践
```

## 七、改进类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 错误纠正 | 从错误中学习 | 命令参数错误 |
| 用户反馈 | 从用户纠正学习 | 输出格式调整 |
| 模式优化 | 从成功经验学习 | 最佳工作流程 |
| 知识扩展 | 从新信息学习 | 新工具使用方法 |

## 八、自动化规则

| 触发条件 | 自动动作 |
|---------|---------|
| 命令失败 | 记录错误、查找解决方案 |
| 用户纠正 | 记录纠正、更新模式 |
| 连续失败 | 提示需要人工干预 |
| 新模式发现 | 提示用户确认 |

## 九、配置要求

```yaml
self_improving:
  storage:
    type: sqlite
    path: ${DATA_DIR}/learning.db
  
  learning:
    auto_apply: true
    confidence_threshold: 0.8
  
  feedback:
    prompt_on_correction: true
    save_user_preferences: true
  
  limits:
    max_errors_per_day: 100
    retention_days: 365
```
