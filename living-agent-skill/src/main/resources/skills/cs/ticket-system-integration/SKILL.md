# Ticket System Integration

> 工单系统集成技能，连接客服工单系统、自动分配、SLA监控

## 一、技能描述

工单系统集成技能使客服部数字员工能够与各类工单系统进行操作，包括：
- 工单创建和分配
- SLA监控和预警
- 工单状态跟踪
- 客户满意度调查

## 二、适用编制

| 编制代码 | 职位 | 主要用途 |
|---------|------|---------|
| C01 | 客服专员 | 工单处理、客户沟通 |
| C02 | 工单处理员 | 工单分配、SLA监控 |

## 三、触发词

- 工单、ticket、工单系统
- Jira、Zendesk、Freshdesk
- SLA、响应时间、处理时间
- 客户满意度、NPS

## 五、使用示例

### 5.1 创建工单

```
用户: 为客户张三创建一个技术支持工单

技能执行:
1. 验证客户信息
2. 创建工单，填写问题描述
3. 自动分配给对应团队
4. 发送确认邮件给客户
5. 返回工单号
```

### 5.2 查询SLA状态

```
用户: 查看今天SLA违规的工单

技能执行:
1. 查询所有进行中工单
2. 计算SLA剩余时间
3. 筛选即将超时的工单
4. 生成预警报告
```

### 5.3 批量分配工单

```
用户: 将待处理工单分配给客服团队

技能执行:
1. 获取待分配工单列表
2. 按技能匹配客服人员
3. 考虑工作负载均衡
4. 执行分配
5. 发送通知
```

## 六、SLA配置

### 6.1 SLA等级

| 等级 | 响应时间 | 解决时间 | 适用场景 |
|------|---------|---------|---------|
| P1 紧急 | 15分钟 | 4小时 | 系统故障、数据丢失 |
| P2 高 | 1小时 | 8小时 | 功能异常、影响业务 |
| P3 中 | 4小时 | 24小时 | 功能咨询、使用问题 |
| P4 低 | 8小时 | 72小时 | 建议反馈、小问题 |

### 6.2 SLA监控规则

| 规则 | 触发条件 | 动作 |
|------|---------|------|
| 即将超时 | 剩余时间<25% | 黄色预警 |
| 即将超时 | 剩余时间<10% | 橙色预警+主管通知 |
| 已超时 | 超过SLA | 红色告警+升级处理 |

## 八、自动化规则

| 触发条件 | 自动动作 |
|---------|---------|
| 新工单创建 | 自动分类、分配 |
| SLA即将超时 | 发送提醒给处理人 |
| 工单解决 | 发送满意度调查 |
| 满意度<3分 | 创建跟进工单 |
| 重复问题 | 关联知识库文章 |

## 十、配置要求

```yaml
ticket:
  platform: zendesk  # zendesk/freshdesk/jira
  
  zendesk:
    subdomain: ${ZENDESK_SUBDOMAIN}
    api_token: ${ZENDESK_API_TOKEN}
  
  sla:
    p1_response: 15m
    p1_resolution: 4h
    p2_response: 1h
    p2_resolution: 8h
    p3_response: 4h
    p3_resolution: 24h
    p4_response: 8h
    p4_resolution: 72h
  
  auto_assign:
    enabled: true
    strategy: round_robin  # round_robin/skill_based/load_balance
  
  notification:
    channels: [email, dingtalk, sms]
```
