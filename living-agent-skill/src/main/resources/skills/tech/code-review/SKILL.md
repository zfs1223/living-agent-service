# Code Review

> 代码审查技能，自动化代码质量检查、安全漏洞扫描、最佳实践建议

## 一、技能描述

代码审查技能使技术部数字员工能够执行自动化代码审查，包括：
- 代码质量检查（SonarQube、ESLint）
- 安全漏洞扫描（CodeQL、Snyk）
- 代码规范检查（Prettier、Checkstyle）
- 最佳实践建议

## 二、适用编制

| 编制代码 | 职位 | 主要用途 |
|---------|------|---------|
| T01 | 代码审查员 | PR审查、代码质量检查 |
| T07 | 安全工程师 | 安全漏洞扫描 |
| T09 | 前端工程师 | 前端代码规范检查 |
| T10 | 后端工程师 | 后端代码规范检查 |

## 三、触发词

- 代码审查、code review、PR审查

- SonarQube、ESLint、CodeQL

## 四、检查维度

### 4.1 代码质量

| 维度 | 检查项 | 工具 |
|------|--------|------|
| 可维护性 | 圈复杂度、代码重复 | SonarQube |
| 可读性 | 命名规范、注释覆盖 | ESLint/Checkstyle |
| 测试覆盖 | 单元测试覆盖率 | JaCoCo/Istanbul |
| 技术债务 | 代码异味、待办事项 | SonarQube |

### 4.2 安全检查

| 维度 | 检查项 | 工具 |
|------|--------|------|
| 注入漏洞 | SQL注入、XSS、命令注入 | CodeQL |
| 敏感数据 | 硬编码密钥、敏感信息泄露 | Snyk |
| 依赖安全 | 第三方库漏洞 | OWASP Dependency Check |
| 配置安全 | 不安全配置 | Checkov |

### 4.3 规范检查

| 语言 | 工具 | 配置文件 |
|------|------|---------|
| Java | Checkstyle | checkstyle.xml |
| JavaScript/TypeScript | ESLint | .eslintrc.js |
| Python | Pylint | .pylintrc |
| Go | golangci-lint | .golangci.yml |

## 五、使用示例

### 5.1 PR代码审查

```
用户: 审查 PR #123

技能执行:
1. 获取PR变更文件列表
2. 运行ESLint/SonarQube分析
3. 检查安全漏洞
4. 生成审查报告
5. 发布审查评论到PR
```

### 5.2 安全漏洞扫描

```
用户: 扫描项目的安全漏洞

技能执行:
1. 运行CodeQL分析
2. 检查依赖漏洞
3. 扫描敏感信息泄露
4. 生成安全报告
5. 标记严重程度
```

### 5.3 代码规范检查

```
用户: 检查代码是否符合规范

技能执行:
1. 运行ESLint/Prettier
2. 检查命名规范
3. 检查注释覆盖
4. 生成修复建议
```

## 六、审查报告模板

```markdown
# 代码审查报告

## 概览
- 审查范围: PR #123
- 文件数量: 15
- 代码行数: +500/-200

## 质量评分
| 维度 | 评分 | 状态 |
|------|------|------|
| 可维护性 | B | ✅ 通过 |
| 安全性 | A | ✅ 通过 |
| 测试覆盖 | C | ⚠️ 需改进 |
| 规范性 | A | ✅ 通过 |

## 问题列表
| 级别 | 数量 | 示例 |
|------|------|------|
| 🔴 阻断 | 0 | - |
| 🟡 警告 | 3 | 未使用的变量 |
| 🔵 建议 | 5 | 可简化的条件 |

## 建议
1. 增加单元测试覆盖 (当前: 45%, 目标: 80%)
2. 提取重复代码到公共方法
3. 添加接口文档注释
```

## 七、自动化规则

| 触发条件 | 自动动作 |
|---------|---------|
| PR创建 | 自动运行代码审查 |
| 发现阻断问题 | 阻止合并，通知作者 |
| 发现安全问题 | 标记为阻塞，通知安全团队 |
| 审查通过 | 自动批准，允许合并 |

## 八、配置要求

```yaml
code_review:
  sonarqube:
    url: ${SONAR_URL}
    token: ${SONAR_TOKEN}
  
  codeql:
    database: ${CODEQL_DB}
    queries: security-and-quality
  
  eslint:
    config: .eslintrc.js
    fix: true
  
  snyk:
    token: ${SNYK_TOKEN}
    org: ${SNYK_ORG}
```

## 九、质量门禁

| 指标 | 阻断阈值 | 警告阈值 |
|------|---------|---------|
| 覆盖率 | < 60% | < 80% |
| 重复率 | > 5% | > 3% |
| 技术债务 | > 1h | > 30min |
| 安全漏洞 | 任何高危 | 任何中危 |
| 代码异味 | > 10 | > 5 |
