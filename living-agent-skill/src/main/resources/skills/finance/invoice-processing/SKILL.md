# Invoice Processing

> 发票处理技能，发票识别、验真、报销流程自动化

## 一、技能描述

发票处理技能使财务部数字员工能够自动处理发票相关任务，包括：
- 发票OCR识别
- 发票真伪验证
- 报销流程自动化
- 发票归档管理

## 二、适用编制

| 编制代码 | 职位 | 主要用途 |
|---------|------|---------|
| F02 | 报销审核员 | 发票审核、验真 |
| F01 | 财务会计 | 发票入账、归档 |

## 三、触发词

- 发票、invoice、发票识别
- 报销、报销审批
- 发票验真、发票查验
- OCR、发票扫描

## 四、功能模块

### 4.1 发票识别

| 功能 | 说明 |
|------|------|
| 图片识别 | OCR识别发票图片 |
| PDF识别 | 解析PDF发票 |
| 信息提取 | 提取发票代码、号码、金额等 |
| 结构化输出 | 输出JSON格式发票信息 |

### 4.2 发票验真

| 功能 | 说明 |
|------|------|
| 税务接口 | 对接国家税务总局API |
| 真伪验证 | 验证发票真伪 |
| 状态查询 | 查询发票状态（有效/作废/红冲） |
| 重复检查 | 检查发票是否重复报销 |

### 4.3 报销流程

| 功能 | 说明 |
|------|------|
| 申请创建 | 创建报销申请 |
| 自动审批 | 低金额自动审批 |
| 人工审批 | 高金额转人工审批 |
| 支付处理 | 审批通过后支付 |

## 五、发票类型支持

| 类型 | 识别 | 验真 |
|------|------|------|
| 增值税普通发票 | ✅ | ✅ |
| 增值税专用发票 | ✅ | ✅ |
| 电子发票 | ✅ | ✅ |
| 火车票 | ✅ | ✅ |
| 出租车票 | ✅ | ⚠️ |
| 航空行程单 | ✅ | ✅ |
| 定额发票 | ✅ | ❌ |

## 六、使用示例

### 6.1 发票识别

```
用户: 识别这张发票图片

技能执行:
1. OCR识别发票内容
2. 提取关键信息
3. 结构化输出
4. 返回发票数据
```

### 6.2 发票验真

```
用户: 验证这张发票的真伪

技能执行:
1. 提取发票代码和号码
2. 调用税务API验证
3. 返回验证结果
4. 记录验证日志
```

### 6.3 创建报销

```
用户: 为这张发票创建报销申请

技能执行:
1. 识别发票信息
2. 验证发票真伪
3. 检查重复报销
4. 创建报销申请
5. 发送审批通知
```

## 八、报销审批规则

| 金额范围 | 审批流程 |
|---------|---------|
| < 500元 | 自动审批 |
| 500-5000元 | 部门主管审批 |
| 5000-50000元 | 部门主管 + 财务审批 |
| > 50000元 | 部门主管 + 财务 + 总经理审批 |

## 九、配置要求

```yaml
invoice:
  ocr:
    provider: baidu  # baidu/aliyun/tencent
    api_key: ${OCR_API_KEY}
  
  verification:
    api_url: https://inv-veri.chinatax.gov.cn
    app_key: ${TAX_APP_KEY}
  
  reimbursement:
    auto_approve_threshold: 500
    require_original: true
  
  storage:
    type: minio
    bucket: invoices
    retention_days: 3650
```
