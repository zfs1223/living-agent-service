# Performance Management

> 绩效管理技能，绩效考核、目标管理、员工发展跟踪

## 一、技能描述

绩效管理技能使人力资源部数字员工能够管理员工绩效相关任务，包括：
- 目标设定和跟踪（OKR/KPI）
- 绩效考核流程管理
- 360度反馈收集
- 绩效报告生成

## 二、适用编制

| 编制代码 | 职位 | 主要用途 |
|---------|------|---------|
| H02 | 绩效管理员 | 绩效考核、报告生成 |
| H01 | 招聘专员 | 入职绩效目标设定 |

## 三、触发词

- 绩效、考核、KPI、OKR
- 目标管理、绩效评估
- 360度反馈、评价
- 绩效报告、绩效面谈

## 四、功能模块

### 4.1 目标管理

| 功能 | 说明 |
|------|------|
| OKR设定 | 季度/年度目标设定 |
| 目标分解 | 部门目标分解到个人 |
| 进度跟踪 | 目标完成进度监控 |
| 对齐检查 | 目标上下对齐验证 |

### 4.2 绩效考核

| 功能 | 说明 |
|------|------|
| 考核周期 | 月度/季度/年度考核 |
| 自评 | 员工自我评价 |
| 上级评价 | 直属上级评价 |
| 同事评价 | 360度反馈 |
| 综合评分 | 加权计算最终得分 |

### 4.3 反馈管理

| 功能 | 说明 |
|------|------|
| 反馈请求 | 发送反馈请求邮件 |
| 匿名收集 | 匿名收集同事反馈 |
| 汇总分析 | 反馈内容汇总分析 |
| 改进建议 | 生成改进建议 |

## 五、使用示例

### 5.1 创建OKR

```
用户: 为技术部创建Q2的OKR

技能执行:
1. 获取公司级OKR
2. 生成部门OKR模板
3. 分解到个人目标
4. 发送确认邮件
5. 创建跟踪看板
```

### 5.2 发起绩效考核

```
用户: 发起技术部Q1绩效考核

技能执行:
1. 创建考核任务
2. 发送自评通知
3. 收集上级评价
4. 发送360度反馈请求
5. 生成考核报告
```

### 5.3 生成绩效报告

```
用户: 生成技术部Q1绩效报告

技能执行:
1. 汇总考核数据
2. 计算绩效分布
3. 识别高绩效/低绩效员工
4. 生成改进建议
5. 发送报告给管理层
```

## 七、考核流程

```
考核周期开始
    │
    ├── 第1周: 目标确认
    │       └── 员工确认考核目标
    │
    ├── 第2周: 自评
    │       └── 员工完成自我评价
    │
    ├── 第3周: 上级评价
    │       └── 直属上级完成评价
    │
    ├── 第4周: 360度反馈
    │       └── 收集同事反馈
    │
    ├── 第5周: 综合评定
    │       └── 计算综合得分
    │
    └── 第6周: 绩效面谈
            └── 上级与员工面谈
```

## 八、绩效等级分布

| 等级 | 分数范围 | 比例控制 | 说明 |
|------|---------|---------|------|
| S | 95-100 | ≤5% | 卓越 |
| A | 85-94 | 20-30% | 优秀 |
| B | 70-84 | 40-50% | 良好 |
| C | 60-69 | 15-25% | 合格 |
| D | <60 | ≤5% | 需改进 |

## 九、配置要求

```yaml
performance:
  cycle: quarterly  # monthly/quarterly/yearly
  
  scoring:
    self_weight: 0.2
    manager_weight: 0.4
    peer_weight: 0.2
    objective_weight: 0.2
  
  distribution:
    S: 0.05
    A: 0.25
    B: 0.45
    C: 0.20
    D: 0.05
  
  notification:
    reminder_days: [7, 3, 1]
```
