# Doc Coauthoring

> 文档协作编写技能，结构化文档创作流程

## 一、技能描述

文档协作技能使行政部数字员工能够引导用户完成结构化文档创作，包括：
- 技术文档编写
- 方案提案撰写
- 决策文档起草
- 规格说明书编写

## 二、适用编制

| 编制代码 | 职位 | 主要用途 |
|---------|------|---------|
| A01 | 行政助理 | 文档整理、会议纪要 |
| A02 | 文档管理员 | 文档归档、版本管理 |
| A03 | 文案创作员 | 方案撰写、报告编写 |

## 三、触发词

- 写文档、写方案、起草
- PRD、设计文档、决策文档
- RFC、规格说明
- 文档协作、coauthoring

## 四、工作流程

### 4.1 三阶段流程

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    文档协作三阶段流程                                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  【阶段一：上下文收集】                                                        │
│  ├── 文档类型确认                                                            │
│  ├── 目标读者分析                                                            │
│  ├── 预期影响定义                                                            │
│  └── 相关信息收集                                                            │
│                                                                             │
│  【阶段二：内容精炼】                                                          │
│  ├── 结构设计                                                                │
│  ├── 逐节编写                                                                │
│  ├── 迭代修改                                                                │
│  └── 整体优化                                                                │
│                                                                             │
│  【阶段三：读者测试】                                                          │
│  ├── 预测读者问题                                                            │
│  ├── 新视角审查                                                              │
│  ├── 问题修复                                                                │
│  └── 最终确认                                                                │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.2 文档类型

| 类型 | 说明 | 典型结构 |
|------|------|---------|
| 技术规格 | 技术实现方案 | 背景/目标/方案/风险/时间线 |
| 决策文档 | 技术决策记录 | 背景/选项/决策/影响/后续 |
| 提案 | 项目提案 | 问题/方案/收益/资源/计划 |
| PRD | 产品需求文档 | 背景/需求/功能/非功能/时间线 |

## 五、使用示例

### 5.1 创建技术文档

```
用户: 帮我写一个技术方案文档

技能执行:
1. 询问文档类型和目标读者
2. 收集背景信息和需求
3. 设计文档结构
4. 逐节编写内容
5. 迭代优化
6. 读者测试
```

### 5.2 编写决策文档

```
用户: 记录我们选择PostgreSQL的决定

技能执行:
1. 收集决策背景
2. 列出备选方案
3. 分析各方案优劣
4. 记录最终决策
5. 说明影响和后续行动
```

## 六、文档模板

### 6.1 技术规格模板

```markdown
# [功能名称] 技术规格

## 背景
[为什么需要这个功能]

## 目标
[这个功能要达成什么]

## 技术方案
[具体实现方案]

### 架构设计
[系统架构图和说明]

### 数据模型
[数据结构设计]

### API设计
[接口定义]

## 风险评估
[潜在风险和应对措施]

## 时间线
[开发计划]
```

### 6.2 决策文档模板

```markdown
# [决策标题]

## 状态
[提议/已批准/已废弃]

## 背景
[决策背景]

## 决策
[决策内容]

## 备选方案
[考虑过的其他方案]

## 影响
[决策的影响范围]

## 后续行动
[需要跟进的事项]
```

## 七、质量检查

### 7.1 内容检查

| 检查项 | 说明 |
|--------|------|
| 完整性 | 是否涵盖所有必要内容 |
| 准确性 | 信息是否准确无误 |
| 一致性 | 内容是否前后一致 |
| 清晰度 | 表达是否清晰易懂 |

### 7.2 读者测试问题

1. 这个文档解决了什么问题？
2. 有哪些内容是模糊的？
3. 文档假设读者已知道什么？
4. 有没有内部矛盾？

## 八、协作技巧

| 技巧 | 说明 |
|------|------|
| 分节编写 | 逐节完成，避免一次性写完 |
| 迭代优化 | 多轮修改，逐步完善 |
| 读者视角 | 从读者角度审查 |
| 精简原则 | 删除不必要的内容 |
