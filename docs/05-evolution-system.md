## 二、信号检测机制

### 2.1 信号来源

| 来源 | 说明 | 示例 |
|------|------|------|
| **对话分析** | 从对话中提取 | 用户反馈"这个回答不对" |
| **日志分析** | 从执行日志提取 | 工具调用失败 |
| **指标分析** | 从性能指标提取 | 响应时间过长 |
| **用户反馈** | 直接用户反馈 | 点赞/点踩 |

### 2.2 信号提取器

```java
public interface SignalExtractor {
    List<EvolutionSignal> extractFromConversation(Conversation conv);
    List<EvolutionSignal> extractFromLogs(LogEntry[] logs);
    List<EvolutionSignal> extractFromMetrics(MetricData[] metrics);
    List<EvolutionSignal> extractFromUserFeedback(Feedback feedback);
}
```

### 2.3 信号类型与触发条件

| 信号类型 | 触发条件 | 强度 | 推荐策略 |
|---------|---------|------|---------|
| **ERROR** | 错误关键词、异常堆栈、失败消息 | 0.8 | REPAIR |
| **PERFORMANCE** | 响应超时、性能下降、资源紧张 | 0.6 | OPTIMIZE |
| **OPPORTUNITY** | 优化建议、改进机会、用户需求 | 0.5 | OPTIMIZE |
| **CAPABILITY_GAP** | 工具失败、技能缺失、能力不足 | 0.7 | INNOVATE |
| **STABILITY** | 内存过高、系统不稳定 | 0.9 | DEFER |

### 3.2 决策考虑因素

| 因素 | 权重 | 说明 |

| 信号强度 | 0.3 | 信号的重要性 |

## 四、执行器

### 4.1 执行策略

| 策略 | 说明 | 执行内容 |
|------|------|---------|
| **REPAIR** | 修复现有技能 | 修改技能代码、更新配置、修复Bug |
| **OPTIMIZE** | 优化技能性能 | 改进算法、缓存优化、并行化 |
| **INNOVATE** | 创造新技能 | 生成新技能代码、注册新技能、测试验证 |
| **DEFER** | 延迟处理 | 记录信号、等待更多证据、定时重试 |
| **ESCALATE** | 上报人工 | 生成报告、通知管理员、等待决策 |

## 五、熔断器机制

### 5.1 熔断状态

| 状态 | 说明 | 触发条件 |
|------|------|---------|
| **CLOSED** | 正常状态 | 失败率 < 阈值 |
| **OPEN** | 熔断状态 | 失败率 ≥ 阈值 |
| **HALF_OPEN** | 半开状态 | 熔断时间结束，尝试恢复 |

### 5.2 熔断规则

```java
public class EvolutionCircuitBreaker {
    // 熔断阈值
    private static final double FAILURE_THRESHOLD = 0.5;  // 50%失败率触发熔断
    private static final int MIN_CALLS = 5;               // 最少调用次数
    private static final Duration COOLDOWN = Duration.ofMinutes(10);  // 冷却时间
    
    // 状态检查
    public boolean canExecute() {
        return state == CircuitState.CLOSED || 
               (state == CircuitState.HALF_OPEN && tryAcquirePermission());
    }
}
```

## 六、进化记忆

### 6.1 记忆内容

| 类型 | 内容 | 用途 |
|------|------|------|
| **SignalHistory** | 历史信号记录 | 模式识别、趋势分析 |
| **DecisionLog** | 决策记录 | 决策复盘、策略优化 |
| **ExecutionResult** | 执行结果 | 效果评估、成功率统计 |
| **PersonalityDelta** | 人格变化 | 追踪人格演进 |

### 6.2 记忆使用

```
进化记忆用于:
├── 避免重复错误
├── 识别进化模式
├── 优化决策策略
├── 追踪人格变化
└── 生成进化报告
```

## 七、与运营评判系统的关联

进化效果影响绩效考核：

| 进化指标 | 考核维度 | 说明 |
|---------|---------|------|
| 技能成功率 | 能力成长 | 新技能/修复后技能的成功率 |
| 进化频率 | 学习积极性 | 单位时间内进化次数 |
| 知识贡献 | 知识沉淀 | 晋升到L2/L3的知识数量 |
| 问题解决 | 业务价值 | 通过进化解决的问题数 |
