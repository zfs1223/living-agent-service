# 安全与权限

> 企业级安全与权限管理系统

## 一、权限隔离架构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    权限隔离架构                                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  权限检查在路由之前执行：                                                      │
│  ├── CHAT_ONLY (离职/外来) → 只能到 Qwen3Neuron                              │
│  │   └── 无法访问 MainBrain、部门大脑、企业知识库                             │
│  │                                                                           │
│  ├── LIMITED (试用期/客户/伙伴) → 受限访问                                    │
│  │   └── 可访问: Qwen3Neuron, AdminBrain, CsBrain                            │
│  │                                                                           │
│  ├── DEPARTMENT (在职员工) → 本部门完整功能                                   │
│  │   └── 可访问: Qwen3Neuron, ToolNeuron, 本部门大脑                         │
│  │                                                                           │
│  └── FULL (董事长) → 所有资源                                                │
│      └── 可访问: 所有神经元和大脑 + MainBrain                                 │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 二、访问级别定义

| 级别 | 权限范围 | 可访问资源 |
|------|----------|-----------|
| **CHAT_ONLY** | 仅闲聊，禁止企业资源 | Qwen3-0.6B (闲聊神经元) |
| **LIMITED** | 部分大脑，禁止敏感知识 | Qwen3.5-27B, AdminBrain, CsBrain |
| **DEPARTMENT** | 本部门完整功能 | Qwen3.5-27B, ToolNeuron, 本部门大脑 |
| **FULL** | 所有大脑和工具 | 所有模型 + MainBrain |

## 三、用户身份与权限映射

| 身份 | 访问级别 | 可用模型 | 特殊说明 |
|------|----------|----------|---------|
| 董事长 | FULL | 所有模型 | 可跨部门协调 |
| 在职员工 | DEPARTMENT | Qwen3.5-27B, Qwen3-0.6B | 仅本部门 |
| 试用期员工 | LIMITED | Qwen3.5-27B, Qwen3-0.6B | 受限访问 |
| 离职员工 | CHAT_ONLY | Qwen3-0.6B | 仅闲聊 |
| 外来访客 | CHAT_ONLY | Qwen3-0.6B | 仅闲聊 |
| 客户 | LIMITED | Qwen3.5-27B, Qwen3-0.6B | 受限访问 |
| 合作伙伴 | LIMITED | Qwen3.5-27B, Qwen3-0.6B | 受限访问 |

## 七、安全沙箱

### 7.1 沙箱配置

```java
public class SandboxConfig {
    // 资源限制
    private long maxMemory;           // 最大内存 (MB)
    private long maxCpuTime;          // 最大CPU时间 (ms)
    private long maxExecutionTime;    // 最大执行时间 (ms)
    
    // 网络限制
    private boolean allowNetwork;     // 是否允许网络访问
    private List<String> allowedHosts; // 允许访问的主机
    
    // 文件系统限制
    private boolean allowFileSystem;  // 是否允许文件系统访问
    private List<String> allowedPaths; // 允许访问的路径
    
    // 命令黑名单
    private List<String> commandBlacklist;
}
```

### 7.2 命令黑名单

```
禁止执行的命令:
├── rm -rf /
├── mkfs.*
├── dd if=/dev/zero
├── :(){ :|:& };:
├── wget | curl | fetch (未授权)
└── 其他危险命令
```

## 八、数据安全

### 8.1 敏感数据处理

| 数据类型 | 处理方式 | 存储位置 |
|---------|---------|---------|
| 员工密码 | 哈希存储 (BCrypt) | PostgreSQL |
| 声纹特征 | 向量加密存储 | Qdrant |
| API密钥 | 加密存储 | 环境变量/密钥管理系统 |
| 对话记录 | 分级存储 | SQLite/PostgreSQL |

### 8.2 审计日志

| 事件类型 | 记录内容 | 保留期限 |
|---------|---------|---------|
| 登录/登出 | 用户ID、时间、IP、结果 | 1年 |
| 权限变更 | 操作人、目标用户、变更内容 | 3年 |
| 数据访问 | 访问者、数据类型、操作 | 1年 |
| 系统配置 | 修改人、配置项、旧值/新值 | 3年 |

## 九、合规管理

### 9.1 合规检查项

| 检查项 | 频率 | 处理方式 |
|--------|------|---------|
| 数据保留期限 | 每日 | 自动清理过期数据 |
| 权限审查 | 每月 | 生成报告，人工审核 |
| 访问日志审计 | 实时 | 异常检测，自动告警 |
| 敏感数据扫描 | 每周 | 发现敏感数据，加密处理 |

### 9.2 合规报告

```
合规报告内容:
├── 数据使用情况统计
├── 权限分配情况
├── 安全事件记录
├── 审计日志摘要
└── 改进建议
```
