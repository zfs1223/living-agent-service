## 三、快速启动

### 3.1 本地开发环境

```bash
# 编译
mvn clean package

# 启动
java -jar living-agent-app/target/living-agent-app.jar
```

### 3.2 Docker部署

```bash
# 快速模式 (核心服务)
docker compose up -d

# 完整模式 (含MemOS、Neo4j)
docker compose --profile full up -d
```

### 3.3 访问服务

| 服务 | 地址 |
|------|------|
| WebSocket | `ws://localhost:8382/ws/agent` |
| REST API | `http://localhost:8382/api/` |
| MemOS API | `http://localhost:8381/` |

## 四、运营评判系统

### 4.1 公司级运营指标

| 指标类别 | 具体指标 |
|---------|---------|
| **财务指标** | 营业收入、净利润率、现金流状况、成本控制率、预算执行率 |
| **业务指标** | 订单量/签约额、客户增长率、客户留存率、市场占有率、项目交付率 |
| **效率指标** | 人均产出、资源利用率、流程效率、数字化程度 |
| **风险指标** | 合规风险指数、运营风险指数、财务风险指数、安全风险指数 |
| **创新指标** | 新产品/服务数量、专利/知识产权、技术创新投入比、数字化转型进度 |

### 4.2 部门级运营指标

| 部门 | 核心指标 |
|------|---------|
| **技术部** | 代码提交量/质量、Bug修复率/引入率、项目按时交付率、系统稳定性(SLA)、技术债务处理率 |
| **销售部** | 销售额/签约额、新客户开发数量、客户转化率、销售周期、客户满意度 |
| **人力资源** | 招聘完成率、员工满意度、培训覆盖率、人才流失率、绩效达成率 |
| **财务部** | 财务报表准确率、预算执行率、成本控制率、资金周转率、税务合规率 |
| **客服部** | 工单处理时效、客户满意度、问题解决率、首次响应时间、投诉处理率 |

### 4.3 绩效考核维度

| 维度 | 权重 | 说明 |
|------|------|------|
| **工作产出** | 40% | 任务完成数量和质量 |
| **工作效率** | 20% | 任务完成时效、响应速度 |
| **协作能力** | 15% | 跨部门协作、知识分享 |
| **创新能力** | 15% | 技能进化、知识贡献 |
| **合规表现** | 10% | 安全合规、数据保护 |

## 六、主动预判系统

### 6.1 四大预判器

| 预判器 | 功能 | 示例 |
|--------|------|------|
| **TimePredictor** | 基于时间规律 | 每周五下午4点生成周报 |
| **EventPredictor** | 基于系统事件 | 新员工入职→准备入职流程 |
| **PatternPredictor** | 基于用户行为模式 | 用户登录后通常查看的项目 |
| **RiskPredictor** | 基于风险指标 | 项目延期风险、预算超支风险 |

### 6.2 主动输出流程

```
触发源 → 预判任务匹配 → 大脑决策 → 任务执行 → 输出分发 → 效果追踪
```

## 七、监控告警

### 7.1 监控指标

| 层级 | 指标 |
|------|------|
| **基础设施** | CPU、内存、磁盘、网络 |
| **应用服务** | 响应时间、吞吐量、错误率 |
| **业务指标** | 对话量、任务完成率、用户活跃度 |
| **AI模型** | Token消耗、推理延迟、准确率 |

### 7.2 告警级别

| 级别 | 触发条件 | 通知方式 |
|------|---------|---------|
| **P0-紧急** | 服务不可用、数据丢失 | 短信+电话+钉钉 |
| **P1-严重** | 性能严重下降、功能异常 | 钉钉+邮件 |
| **P2-一般** | 性能下降、非核心功能异常 | 钉钉 |
| **P3-提示** | 资源使用率预警 | 邮件 |

## 八、备份与恢复

### 8.1 备份策略

| 数据类型 | 备份频率 | 保留期限 |
|---------|---------|---------|
| PostgreSQL | 每日全量 | 30天 |
| Qdrant | 每周全量 | 14天 |
| Redis | 每小时RDB | 7天 |
| 配置文件 | 每次变更 | 永久 |

### 8.2 恢复流程

```
1. 评估故障范围和影响
2. 选择恢复点 (RPO)
3. 执行数据恢复
4. 验证数据完整性
5. 恢复服务
6. 验证服务可用性
```
