# Living Agent Service - 生命智能体自治系统

> 带生命的AI智能体自治系统 - 自主进化、赚钱驱动、持续成长

***

## 一、项目概述

### 1.1 项目定位

本项目从 `dialogue-service` 演化而来，专注于构建**生命智能体自治系统**。核心设计理念：

- **神经元群聊模式**：每个智能体作为"神经元"，通过通讯管路协作
- **仿脑神经中枢架构**：按大脑功能分区设计智能体职责
- **带生命的智能体**：具备感知、决策、执行、学习、进化、赚钱能力
- **自主进化驱动**：通过赚钱实现经济独立，收益用于硬件升级和技能进化

## 三、核心架构概览

### 3.1 三层LLM架构

```
Layer 1: 主大脑 (MainBrain) - 灵活配置
├── 复杂推理、跨部门协调、战略决策
├── 默认: Qwen3.5-27B (云端API)
├── 可选: Qwen3.5-14B / Qwen3-32B / DeepSeek-V3 等
├── 动态模型选择 (MainBrainModelSelector)
└── 配置: main-brain.model.default / main-brain.model.api-key

Layer 2: 闲聊神经元 (Qwen3Neuron) - Qwen3-0.6B (固定)
├── 日常对话、快速响应、简单任务
├── 独立运行，不参与灵活配置
└── 本地部署，低延迟响应

Layer 3: 工具神经元 (ToolNeuron) - 灵活配置
├── 工具检测、兜底处理、触发进化信号
├── 默认: Qwen3.5-2B (推荐，支持多模态)
├── 备选: BitNet-1.58-3B (低资源环境)
├── 动态模型选择 (ToolNeuronModelSelector)
└── 原生多模态能力 (Qwen3.5-2B)
```

> **模型配置策略**：
> - **Layer 2 (Qwen3Neuron)**: 固定使用 Qwen3-0.6B，独立运行，不参与灵活配置
> - **Layer 1 (MainBrain)**: 通过 `MainBrainModelSelector` 支持多种云端模型
> - **Layer 3 (ToolNeuron)**: 通过 `ToolNeuronModelSelector` 根据硬件资源自动选择
> - **新建数字员工**: 使用本地部署模型，参考 Clawith-main 的 LLM 客户端设计

#### 3.1.1 模型选择器配置

**MainBrainModelSelector 配置**:
```yaml
main-brain:
  model:
    default: qwen3.5-27b        # 默认模型
    api-key: ${MAIN_BRAIN_API_KEY}
    base-url: ${MAIN_BRAIN_BASE_URL:}  # 可选，自定义API地址
    auto-select: true            # 自动选择（基于任务复杂度）
    available-models:
      - qwen3.5-27b
      - qwen3.5-14b
      - qwen3-32b
      - deepseek-v3
```

**ToolNeuronModelSelector 配置**:
```yaml
tool-neuron:
  model:
    default: qwen3.5-2b          # 默认模型（推荐）
    auto-select: true            # 根据硬件资源自动选择
    memory-threshold-mb: 2048    # 内存阈值
```

> **进化系统**：所有神经元共享 `EvolutionExecutor` + `SkillGenerator` + `KnowledgeEvolver` 实现自我成长能力。

### 3.1.2 Layer 3 模型选择策略

| 模型 | 默认状态 | 内存需求 | 上下文长度 | 多模态 | 适用场景 |
|------|---------|---------|-----------|--------|---------|
| **Qwen3.5-2B** | ✅ 默认 | 4GB | 262K | ✅ | GPU推理、高性能场景 |
| **BitNet-1.58-3B** | 备选 | 1GB | 4K | ❌ | CPU推理、资源受限场景 |

**自动切换条件**：
- 内存 ≥ 4GB + CPU ≥ 4核 + CPU负载 < 80% → Qwen3.5-2B
- 内存 < 4GB 或 CPU负载 ≥ 80% → BitNet-1.58-3B

**配置示例**：
```yaml
tool-neuron:
  model:
    default: qwen3.5-2b
    auto-select: true
    memory-threshold-mb: 2048
```

## 七、快速开始

### 7.1 环境要求

- Java 21+
- Maven 3.9+
- Rust 1.75+
- Python 3.11+
- PostgreSQL 15+ (可选)
- Qdrant 1.7+ (可选)

### 7.2 启动服务

```bash
# 编译
mvn clean package

# 启动
java -jar living-agent-app/target/living-agent-app.jar
```

### 7.3 Docker部署

```bash
# 快速模式 (核心服务)
docker compose up -d

# 完整模式 (含MemOS、Neo4j)
docker compose --profile full up -d
```

### 7.4 访问服务

- WebSocket: `ws://localhost:8382/ws/agent`
- REST API: `http://localhost:8382/api/`
- MemOS API: `http://localhost:8381/`
